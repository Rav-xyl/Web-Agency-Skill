# Contributing to Web Agency Skill (WSA)

First off, thank you for considering contributing to WSA! It's people like you who make this framework a better tool for the entire solo-dev and agency community.

---

## 🎭 The Philosophy
WSA is a **Senior-Partner-in-a-Box**. Every contribution should uphold the values of:
1. **Architectural Integrity**: Normalized schemas, type safety, and transactional reliability.
2. **High-Aura UI/UX**: Cinematic motion, fluid typography, and premium aesthetics.
3. **Zero Placeholder Tolerance**: No slop. Production-ready or don't commit.

---

## 🛠️ How Can I Contribute?

### 1. Improving Reference Modules
The `references/` folder contains the core knowledge. If you have a professional-grade pattern for a specific stack or vertical (e.g., "Advanced E-commerce SEO" or "Multi-tenant DB Hardening"), we want it.
- **Criteria**: Must be agnostic, code-heavy, and production-tested.

### 2. Proposing New Commands
Commands in `commands/` define the lifecycle logic. If you identify a gap in the GSD workflow (e.g., a "Migration Audit" or "Growth Analytics Sync"), propose it!

### 3. Reporting Bugs
Use the GitHub Bug Report template to report any inconsistencies in the skill logic.

---

## 🚦 Pull Request Process
1. **Fork the repo** and create your branch from `main`.
2. **Uphold the Standards**: Ensure your additions don't introduce placeholders or "junior-level" patterns.
3. **Self-Audit**: Run a placeholder scan on your changes before submitting.
4. **Documentation**: If you're adding a reference, update `SKILL.md` to link it.
5. **PR Description**: Explain the *why* behind your architectural choice.
